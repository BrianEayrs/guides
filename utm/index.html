## What UTMs are (context for you)
UTM parameters are tags added to the end of a destination URL. When someone clicks the ad, those tags travel with them and get recorded by my website, analytics, and CRM. They let me answer "which ad did this lead come from?" instead of guessing.

There are five standard parameters:
- utm_source — where the traffic comes from (the platform). Example: facebook
- utm_medium — the type of traffic. Example: paid_social
- utm_campaign — which campaign. Example: summer_launch
- utm_content — which specific ad or creative. Example: hook_a_video
- utm_term — optional, usually the audience or keyword. Example: lookalike_1pct

## Naming rules (enforce these on every value)
- All lowercase.
- No spaces. Use underscores ( _ ) between words.
- No special characters ( no & ? = # / ' " ).
- Be consistent. "facebook" and "Facebook" and "fb" are three different sources to analytics, so pick one and never change it.
- Keep values short but readable, so I can tell them apart in a report six months from now.

## Meta dynamic parameters (use these so I never hand-edit per ad)
Meta can auto-fill values at click time. Prefer these where they fit, so one URL template works across a whole campaign:
- {{campaign.name}} — fills in the campaign name
- {{adset.name}} — fills in the ad set (usually the audience)
- {{ad.name}} — fills in the ad (usually the creative/hook)
- {{placement}} — fills in where it showed (feed, story, reel, etc.)

A good default template using these:
utm_source=facebook&utm_medium=paid_social&utm_campaign={{campaign.name}}&utm_content={{ad.name}}&utm_term={{adset.name}}

Note: for Meta's dynamic values to read cleanly in reports, name my campaigns, ad sets, and ads in lowercase_with_underscores too.

What UTMs actually do

A UTM is a small tag added to the end of a link. When someone clicks your ad, those tags ride along with them and get recorded by your website, your analytics, and your CRM. Without them, every click looks the same. With them, you can tell that this lead came from the reel hook running to the cold audience, and that one came from a retargeting ad.

The five parameters

  • utm_source — the platform the traffic came from. Example: facebook
  • utm_medium — the type of traffic. Example: paid_social
  • utm_campaign — which campaign. Example: summer_launch
  • utm_content — the specific ad or creative. Example: hook_a_video
  • utm_term — optional. Usually the audience or keyword. Example: lookalike_1pct

Naming rules

Analytics reads your text literally, so consistency is everything.

  • All lowercase.
  • No spaces. Use underscores between words.
  • No special characters like & ? = # / or quotes.
  • Pick one spelling and never change it. facebook, Facebook, and fb count as three different sources.
  • Keep values short but readable, so they still make sense in a report six months from now.

Let Meta fill it in for you

Instead of hand-editing a URL for every ad, Meta can auto-fill the values at click time. Use these placeholders and one template covers a whole campaign:

  • {{campaign.name}} — the campaign name
  • {{adset.name}} — the ad set, usually your audience
  • {{ad.name}} — the ad, usually your creative or hook
  • {{placement}} — where it showed: feed, story, reel

For these to read cleanly in reports, name your campaigns, ad sets, and ads in lowercase_with_underscores too.

Where it goes in Ads Manager

At the ad level, put your clean destination URL (no parameters) in the Website URL field. Put the UTM string in the URL parameters field. Or paste the full URL with parameters straight into Website URL. Do one or the other, never both, or the tags get doubled up.

Set up once, name things consistently, and every click tells you where it came from.
